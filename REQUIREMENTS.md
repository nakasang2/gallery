# HAKONIWA — 3Dバーチャルギャラリー 要件定義(ライト版)

> あなたの作品が、空間になる。

## 1. コンセプト

InstagramやXでアートを発表しているクリエイターが、**自分専用の3Dギャラリー空間**に作品を展示し、URLひとつで世界に公開できるプラットフォーム。
タイムラインに流れて消える「投稿」ではなく、来場者が歩いて鑑賞する「展覧会」として作品を見せる。

- 名称(仮): **HAKONIWA(箱庭)** — 自分だけの小さな世界を作り込む、という意味を込めて

## 2. ターゲットユーザー

| ユーザー | 課題 | HAKONIWAが提供する価値 |
|---|---|---|
| イラストレーター / 画家 | SNSでは作品が流れて埋もれる | 常設の個展空間・ポートフォリオ |
| 写真家 | 一覧グリッドでは世界観が伝わらない | 順路・照明・空間演出で文脈ごと見せる |
| 美大生 / 卒展主催者 | 物理会場のコスト・期間の制約 | 無料で無期限のオンライン展示 |
| アートファン(来場者) | 気軽にギャラリーに行けない | ブラウザだけで入場、没入体験 |

## 3. 機能要件

### MVP(フェーズ1)— まずこれだけで成立させる

- **アカウント**: 登録 / ログイン(メール + SNS認証)
- **作品アップロード**: 画像(JPG/PNG/WebP)+ タイトル・説明・制作年・タグ
- **ギャラリー作成**: テンプレート空間を選び、壁面に作品をドラッグ配置
- **3D鑑賞体験(来場者)**:
  - ブラウザのみで動作(アプリ・プラグイン不要)
  - WASD / ドラッグ / タップで移動・視点操作
  - 作品クリックで拡大 + キャプション表示
- **公開・共有**: 固有URL発行(`hakoniwa.app/@username/exhibition`)、OGP対応
- **レスポンシブ**: PC / スマホ両対応

### フェーズ2(公開後に拡張)

- いいね・コメント・芳名帳(来場記録)
- 空間カスタマイズ(壁色・床材・照明・BGM)
- 同時接続マルチプレイ(他の来場者のアバターが見える)
- 作品販売 / デジタル原画のEC連携
- アクセス解析(来場者数・滞在時間・注目作品)
- ギャラリー横断の「街」— 特集・キュレーション導線

## 4. 非機能要件

- **パフォーマンス**: 初回ロード3秒以内(空間は遅延ロード)、モバイル30fps以上
- **対応環境**: 最新のChrome / Safari / Edge / Firefox(WebGL 2.0)
- **画像処理**: アップロード時に自動リサイズ・テクスチャ最適化
- **セキュリティ**: 画像の無断DL抑止(右クリック対策は気休め程度と明記)、通報機能
- **アクセシビリティ**: 2D一覧フォールバック(WebGL非対応環境・スクリーンリーダー向け)

## 8. 追加要件(v0.2 〜)

プロトタイプ初版へのフィードバックから決定した拡張要件。

1. **回遊性**: モバイル / PC それぞれの操作系で直感的にギャラリーを回遊できる
   - → v0.2で実装: タッチ端末にバーチャルジョイスティック(+従来の床タップ移動)、全端末に作品を順に巡る「順路ツアー」。任意の操作でツアーは中断
2. **出展フロー**: Instagram等に投稿済みのアートから選んでギャラリーに出展できる。作品のアスペクト比に合わせて額装・配置が自動調整される
   - 注: Instagram Basic Display APIは2024年12月に廃止済み。公式連携はGraph API(ビジネス/クリエイターアカウント)が必要なため、プロトタイプではURL貼り付け・アップロードで代替し、連携UIはモックとする
   - → v0.2で実装: 画像アップロード(長辺1280pxへ自動縮小しlocalStorageへ保存)/ 画像URL貼り付け / IG連携モック。縦横比から額装サイズを自動算出(極端なパノラマは幅2.6mで頭打ち)
3. **空間カスタマイズ**: ギャラリーのデザイン(壁・床・照明のテーマ)とレイアウト(部屋の形・順路)を任意に変更できる
   - → v0.2で実装: テーマ3種(シック / ホワイトキューブ / ノワール)× レイアウト3種(ワンルーム / 回廊 / 中央壁)。「空間を編集」パネルから即時切替
4. **額装カスタマイズ**: アートのフレームデザイン(色・材質・マット紙)を任意に変更できる
   - → v0.2で実装: 額装5種(墨 / 金 / 白 / 木 / なし=キャンバス張り)。全体一括と作品ごとの個別指定(鑑賞パネルから)の両対応
5. **没入感**: ローポリ感を脱し、実在のギャラリーに近い質感・光・空気感を実現する
   - → v0.2で実装: PBRテクスチャ、環境マップ、静的ベイク影、面取り額縁、AO + ブルーム + SMAA(モバイルは自動で軽量化)
6. **展示テンプレートと空間軸の拡張**(v0.35で実装):
   - テンプレート4種(テーマ/レイアウト/額装/吊し方/キャプションを一括適用する出発点)
   - 吊し方3種(レールワイヤー / 直付け / 棚置き)・キャプション3種(横 / 下 / なし)を新軸として追加
   - テーマは推奨値(額装・吊し方・キャプション)を持ち、テーマ選択時に適用。後から個別変更可(ハード結合ではない)
   - 額装・マット(有無+色: 額縁既定/なし/白/アイボリー/グレー/黒)・吊し方・キャプションは
     **作品単位でも上書き可**(鑑賞パネル / ダッシュボードのプレビュー横の構造化パネル。
     ギャラリー既定と同じ値を選ぶと上書き解除 → 以後テーマに追従)(mat: migration 0012)
   - 額縁は**パラメトリック指定**(有無スイッチ × 素材3種 × 色8色サークル × 太さスライダー3〜15cm)。
     プリセット5種は curated な組み合わせとして共存。カスタム値は `c:素材:色:太さmm` キーとして
     既存のテキストカラム(frame_default / frame_override)をそのまま流れる — migration不要
   - **作品のタイトル・キャプション文はダッシュボードで編集**でき、ネームプレートに描画される
     (キャプションがあればタグの代わりに最大2行・省略記号付き。artworks.description 既存カラム利用)
   - **ダッシュボードのGalleryタブは単一ワークベンチ**: 作品ライブラリ(左)+実3Dプレビュー(右)に
     プレート文言 / 作品別デザイン / 部屋全体のテーマ・レイアウトを集約。3D確認は「Preview in 3D」から
   - 縦長レイアウト「Portrait Hall」追加
7. **操作系の改善**(v0.35で実装):
   - ステア型移動(オートヘディング): 左右入力=旋回、前後=歩行で、常に進行方向を向く。床タップも移動方向へ自動で向く
   - ジョイスティックを全端末に表示(PCはマウスで操作)
   - 自分のペースで巡る「‹ 前へ / 次へ ›」ステッパー(移動+正対+解説表示を1タップで)
   - スマホの作品詳細をボトムシート化し、作品を画面上部にフレーミング(詳細が作品を隠さない)

## 9. 今回のプロトタイプの範囲

このリポジトリで作るのは **1と2**:

- ランディングページ(コンセプト提示・世界観)
- 歩ける3Dギャラリーデモ(サンプル作品を展示、移動・鑑賞・作品詳細表示)

認証・アップロード等のバックエンドはプロトタイプでは対象外(モック)。

## 10. 追加要件(v0.4)— アカウントと複数箱庭

> ステータス: **確定**(2026-07-09)。10.8参照。

### 10.1 認証(メール + パスワード)

- **新規登録**: メールアドレス + パスワード(8文字以上)。確認メールのリンクで有効化
- **サインイン**: メールアドレス + パスワード
- 既存の**マジックリンク / Google OAuth は併存**(TBD-1)
- **パスワードリセット**: 「パスワードを忘れた」→ メールのリンク → 再設定画面
- ログイン中の操作: パスワード変更 / メールアドレス変更(要確認メール)/ 表示名・bio編集(既存)
- ログアウト / **アカウント削除**(箱庭・作品・Storageファイルも削除)
- 画面: `/signup` `/signin` `/reset` の専用ページ。LPと `/demo` から導線
- **ゲスト移行**: ゲスト時にlocalStorageへ出展した作品を、登録直後に「アカウントへ取り込みますか?」と提案

### 10.2 箱庭(ギャラリー)の複数化

- **設計は複数箱庭対応、リリース時の上限は1人1個**。上限は将来プランごとに変えられるよう
  プラン変数(10.10)として持つ(DBスキーマは既に複数対応済み。制限はアプリ層で実施)
- 箱庭ごとに保持するもの:
  - タイトル / slug(公開URL用・ユーザー内一意・変更可)/ ステートメント
  - 空間設定: テーマ / レイアウト / 額装 / 吊し方 / キャプション(テンプレート一括適用も可)
  - 配置: どのスロットにどの作品(+作品ごとの額装・吊し方・キャプション上書き。
    未設定 = ギャラリー既定を継承。テーマ変更時は確認の上リセット)(migration 0011)
- **作成フロー**: ダッシュボード「新規作成」→ テンプレート選択(または個別設定)→ 名前入力 → 3Dエディタが開く
- **編集**: 3D空間(現行 `/demo` 体験)をその箱庭のデータで開き、作品の追加・入替・空間設定変更 → **保存**
- **保存と公開**: 保存 = DBへ書き込み。公開トグルON = `/@username/slug` で公開、OFF = 非公開(404)
  - 公開中の箱庭は保存すると即反映(スナップショット方式は廃止)(TBD-3)
- 削除(確認ダイアログ必須)。複製はTBD-4

### 10.3 作品ライブラリ

- 作品は**箱庭から独立したユーザー資産**(現行スキーマ通り)。複数の箱庭で同じ作品を使い回せる
- ダッシュボードに「作品」タブ: 一覧 / アップロード / 削除(どこかの箱庭で使用中なら警告)
- 上限: ストレージ容量 **300MB/ユーザー**(プラン変数)。動画は既存の40MB/本を踏襲

### 10.4 ダッシュボード(`/me`)

- **箱庭一覧**: サムネイル(代表作)/ タイトル / 公開状態バッジ / 更新日時
- 各箱庭の操作: 開く(3Dエディタ)/ 公開ページを見る / URLコピー / 名称変更 / 公開・非公開 / 削除
- 「新規作成」ボタン(上限到達時は理由を表示)
- **プロフィール編集**: username(公開URLの一部)/ 表示名 / bio / アバター(TBD-6)
- 「作品」タブ(10.3)

### 10.5 公開ページ

- `/@username`: プロフィール + **公開中の箱庭一覧**(URL設計に既存・v0.4で実装)
- `/@username/[slug]`: 既存の3D鑑賞ページ(吊し方・キャプション対応済み)
- OGP: 既存方式(代表作画像)を踏襲。代表作の扱いはTBD-7

### 10.6 非機能・安全

- RLS: 既存方針を複数箱庭へ拡張(ownerのみ書き込み / 公開ギャラリーとその配置・作品のみ誰でも読める)
- アップロードのレート・サイズ検証(クライアント + Storageポリシー)
- 公開ページに「問題を報告」導線(まずは mailto、DB化はフェーズ2)
- 利用規約 / プライバシーポリシーページ(一般公開前に必須)
- メール到達性: 当面Supabase標準SMTP(TBD-8)

### 10.7 画面遷移(v0.4後)

## 11. ビジネスモデル・原価・配信アーキテクチャ

> ステータス: **方針(2026-07-11)**。実装(Stripe/プランDB化)は未着手。金額は要検証のたたき台。

### 11.1 基本方針 —— 「できるだけ買い切り」

- **無料で"作って公開する"を完結**させ、口コミの起点にする。作品を人質にしない(公開は無料でも取り上げない)
- 課金は **「もっと・より重く」に対してだけ**。原価の性質でモデルを分ける:
  - **限界コストがほぼゼロのもの**(創作機能・画像スロット)→ **買い切り**(サブスク疲れの創作者に刺さる)
  - **配信コストが変動するもの**(動画)→ **有料+上限管理**(買い切りの無限責任を避ける唯一の例外)
- 原則: **買い切りが成立するのは"在庫ゼロの機能"に対してだけ**。帯域は在庫があるので、配信最適化(11.3)で在庫化コストを潰してから買い切りに載せる

### 11.2 原価構造 —— 主犯は「保管」ではなく「配信(egress)」

| 対象 | 実コストの正体 | 大きさ |
|---|---|---|
| 画像の保管 | ストレージ | **極小**(1作品 ≈ 0.34MB。300MBに約800枚) |
| 画像の配信 | egress(来場ごとの転送) | 配信先次第。R2なら**≒ ゼロ**(11.3) |
| 動画の配信 | egress(再生ごとの転送) | **外れ値**。人気に比例して青天井になり得る |
| 固定基盤 | 認証・DB・計算 | 全ユーザーで按分。無料ユーザー1人あたりは僅少 |

**画像は保管もR2配信もほぼ無料**なので、画像に関しては買い切りが原価的に完全に成立する。**唯一の変動費は動画の配信**。

### 11.3 配信・原価アーキテクチャ(egress対策)

egressは設計でほぼ潰せる。効く順:

1. **ゼロegressストレージへ載せ替え**: 画像・サムネを **Cloudflare R2**(egress $0、保管のみ課金)へ。画像配信コストを実質消滅させる。代替: Bunny.net(帯域激安+画像最適化内蔵)
2. **CDNキャッシュ**: メディアを**コンテンツハッシュのファイル名 + `Cache-Control: immutable, max-age=1年`**。2回目以降の来場はエッジ配信=オリジンegressゼロ(人気ギャラリーほど効く)
3. **見た分だけ配信**: ストリップ/遠景はサムネ(0.04MB)のみ、**フォーカスした作品だけ**高解像度を取得。画像は **WebP/AVIF**(同画質−30〜50%)。3Dテクスチャは **KTX2/Basis** 圧縮
4. **動画は自前配信しない**: **Cloudflare Stream / Bunny Stream / Mux** に載せ、アダプティブビットレート+エッジ配信。既存の近接再生に加え **ポスター先行→接近/操作で初めてストリーム開始** で素通り再生を排除

効果目安: R2移設で画像配信 ≒ ゼロ / CDNで再訪 −90%超 / 遅延+AVIFで初回 −50〜70% / 動画をStream系へで桁減。

### 11.4 買い切り成立の裏付け(原価計算)

- **画像1作品** ≈ 表示用0.3MB + サムネ0.04MB ≈ **0.34MB**
- **Proユーザー(画像20点)** = 6.8MB。R2保管 $0.015/GB月 × 60か月 ≈ **$0.006(1円未満/5年)** + egress $0
  → 一度きりの支払いが**数世紀分の原価をカバー**。画像スロットの買い切りは余裕で持続可能
- **動画**(Cloudflare Stream ≒ 配信1,000分あたり$1): 1分の動画が1,000回再生で約$1。**再生数=人気に比例する変動費**なので、純粋な無制限買い切りは不可 → 上限管理が必須

### 11.5 料金設計(たたき台)

**無料(¥0・永久)**
- 1展示室 / **画像5点**(その部屋の物理キャパ) / 公開ホスティング / プリセットの額縁・レイアウト・テーマ / 芳名帳・基本分析
- 5点は「試して公開」には十分、「作品集として見せる」には少し足りない絶妙なライン(アップグレード動機を残す。10点だと満足して課金理由が消えがち)

**展示室は「部屋」と「作品枠」を分けて売らない —— 1購入=1展示室まるごと**

100作品を1部屋に詰めるのは描画負荷・回遊性の面でも不可能なので、**キャパは部屋ごとの物理上限で頭打ち**にする(既存の `worksPerGallery` がこの役割)。ここで「部屋」と「枠」を別売りにすると、購入者が毎回「これは部屋?枠?」と迷う二重の意思決定になりストレスが生まれる。そこで:

> **展示室を追加する(買い切り・繰り返し購入可)** = 新しい部屋(新URL・新テーマ)+ **その部屋のキャパ上限までの作品枠が最初から込み**

ユーザーが決めるのは「もう1部屋要るかどうか」だけ。中間の「枠だけ買う」選択肢は見せない。

| SKU | 内容 | 目安 |
|---|---|---|
| **展示室を追加**(主役SKU・繰り返し購入可) | 新しい展示室(部屋の物理キャパ、例15点、まで最初から使える) | ¥1,500〜/室 |
| デザインPro(買い切り) | カスタム額縁(素材×色×太さ)/ 追加レイアウト・テンプレート / マット色。**全展示室に効く**横断アドオン | ¥1,500 |
| Proバンドル(買い切り) | 展示室2部屋分 + デザインPro + 独自ドメイン + OGPカスタム + 詳細分析 | ¥4,900 |

**動画(買い切りの唯一の例外)**
- 配信コストが変動するため、以下いずれか:
  - **動画パス(年額)**: 動画展示を有効化。フェアユースの**配信分数上限**付き。「配信コストの維持費」として年更新
  - または **買い切り動画枠 + 配信分数込み**、超過分はトップアップ(従量)
- 上限に達したら停止/ポスター静止に**フォールバック**(課金トラブルで展示が壊れない設計)

### 11.6 買い切りの"無限責任"を塞ぐ安全弁(規約)

- **休眠アーカイブ**: 非アクティブ(例: 2年ログインなし)の公開ギャラリーは非公開化 or 静的スナップショット化する場合がある旨を明記
- **保有と公開の切り分け**(代替案): 「作品の**保有・編集**は永久、3D**公開ホスティング**は年ごとの維持」とすることも可能
- **画像帯域フェアユース**: R2+CDNで実質不要になるが、異常トラフィックへの但し書きは残す

### 11.7 実装マッピング(将来)

- 料金プランは `lib/limits.ts` の **プラン変数(10.10)** に写像する。無料/アドオンをプラン定義+アドオンフラグ(`videoEnabled` 等)へ展開
- **「展示室を追加」= `PLAN.galleries` を+1し、新規 `createGallery` を1回実行**。その部屋の `worksPerGallery` は購入時点のプラン値を継承する(1レコード=1室のキャパ一体管理。将来キャパ改定時、既存室を購入時の値で固定するか全室追従させるかは別途ビジネス判断)
- デザインPro等の横断アドオンは、ユーザーのプランレコードにフラグとして持たせ、**全展示室が参照**する(部屋ごとに再購入させない)
- 決済は Stripe。**買い切り = one-time Checkout(展示室追加は複数回購入可能な同一SKU)、動画パス = 年額サブスク**。購入結果をユーザーのプランレコード(DB)に反映し、`limits.ts` の参照点を差し替える
- 配信最適化(11.3)は課金より先行して入れてよい(無料ユーザーの原価を下げるほど無料枠を広く保てる)
